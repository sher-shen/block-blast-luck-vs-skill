# 游戏规则 + 方块目录

## 规则（用户描述的那款，Block Blast / 1010! 类）
- 8×8 固定棋盘，无重力、不下落。
- 每轮发 **3 个**随机方块，必须 3 个全放完才刷新下一批。
- 放下后填满的**整行/整列立即消除**；其余格子原位不动。
- 当手里还有方块但棋盘上无任何合法位置 → 游戏结束。
- 得分（现行建模，`scoring.py`，**非线性**）：per_cell + multi-clear 三角数 + combo 连击 + all-clear 清盘奖励。
  数值是**假设档（assumed）非游戏实测**；2026-06-06 起另有 `real_approx` 档（社区逆向：bonus 表 + 乘法连击，
  非官方），见 calibrate_scoring.json（real/assumed≈0.71×，同量级）。
  ~~早期线性版"得分 = 累计消除的行+列条数"~~ 已弃用（仅 RESULTS.md/旧数字用过）。

## 方块目录（建模假设）
- **38 种**（含朝向）。把不同旋转朝向算作不同 piece，每种**均匀等概率**。
- 覆盖：1格 / 2格(横竖+两对角线) / 3格(一字/L/对角) / 4格(全套tetromino+2×2) / 5格(一字五+十字) / 6格(2×3) / 9格(3×3)。
- 生成方式：`pieces.py` 里给基础形状，自动生成所有不重复旋转再去重。

## 关键限定
结论强依赖这份目录 + 均匀假设。真实游戏可能：(a) 方块集不同；(b) 概率非均匀；
(c) 自适应 RNG（按棋盘发顺/坑你，如 HATETRIS）。换设定要重算。
**实测（2026-06-06，deal_audit.json）：均匀 38 型确实偏硬**——均值 3.82 格/块、15.8% ≥5 格、7.7% 手含 3×3；
仅去掉 3×3（删 2.6% 概率质量）最强策略 surv 即 ×2.7。真实游戏疑似偏善意发牌 → 均匀假设下的分数/生存是保守值。
