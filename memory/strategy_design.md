# strong 策略怎么做的 + 是不是最优 + 通往最优的路线图

## 我们的 strong 策略是什么
**不是最优，是一个"手工启发式 + 局部搜索"的强玩家。** 三件事：
1. **启发式打分** `heuristic(board)`：`-1.0*已填格子 - 0.5*碎片度`。
   碎片度 = 每个空格四邻被墙/填充挡住的方向数之和（空格越连成片越好）。
2. **整手换序搜索**：枚举 3 个方块的 6 种放置顺序。
3. **每步贪心**：每个方块在所有合法位上选 `消除数*5 + heuristic` 最大的位置。
取 6 种顺序里整手最优的那个落子。

## 有没有现成代码
有。GitHub 上有 Block Blast / 1010! 的开源 solver，主流做法都是：
- 启发式贪心（和我们类似）
- **MCTS（蒙特卡洛树搜索）**
- **强化学习 DQN / PPO**
**但没有一个是"可证明最优"的**——全是"更强的近似"。

## 怎么知道一个策略是不是最优？（关键）
理论上"最优策略"= 解这个 MDP 的最优期望得分，用**动态规划/价值迭代**。
但 8×8 状态空间 = 2^64 棋盘 × 手牌组合，**天文数字，无法精确求解**。所以：

1. **小棋盘精确解做基准**：在 4×4（状态 2^16，可枚举）上用 DP 解出真最优，
   对比我们的启发式差多少 → 若接近，给大棋盘以信心。
2. **技能阶梯 + 收敛判据**（最实用）：做 greedy → 深前瞻 → MCTS → RL 一串越来越强的玩家，
   测每级的平均分和"运气占比"。**当玩家继续变强、分数和运气占比不再变化时，
   就说明已逼近最优**——剩下的方差就是不可消除的纯运气。
3. **上界 bound**：用"事后诸葛"（已知整条序列后离线最优放置）算一个分数上界，
   看在线策略离它多远。

## 对"运气占几成"这个问题的重要洞察
**不需要真最优**就能回答。我们要的是**收敛**：把玩家做得足够强，当"再变强也不涨分"时，
此刻测出的运气占比就是答案。当前 strong 还没到顶 → 预期加 MCTS/RL 后运气占比会**继续升**
（可能到 75%+），因为更强的玩家把"技能能控的部分"榨干，剩下的更纯是运气。

## 已执行：lookahead/前瞻玩家 (fast.py bitboard + compare.py 配对CRN)
- 实现 beam-strong(手内 beam, 连击贯穿) + lookahead(beam候选→flat MC rollout, λ=1, CRN)。
- **关键发现(K=20 配对)**：
  - LA_greedy − strong = −3640±1717 → greedy 做 rollout 基策略**更差**(rollout 回归陷阱)。
  - LA_strongbase − LA_greedy = +2427±1007 → 换 strong 基策略显著回血(证实:基策略须≥被改进者)。
  - LA_strongbase − strong = −1214±1814 → 前瞻与 beam-strong **统计打平** → **收敛**。
- 结论：beam-strong 已近这一启发式族技能天花板，加搜索深度收益递减(收敛证据)。
- 真要逼近 oracle(~12000 vs strong~6100)需**学习型价值函数(RL/DQN)**，非更深搜索。
- 工程教训：bitboard(64位int)必需；rollout 必须 CRN 配对+独立RNG；天花板别循环自定义。
